# 비주얼 렌더링 계획

지금까지(Phase 0~10) 구현된 렌더링 코드를 기준으로 현재 비주얼 스타일을 정리하고,
앞으로 다듬어 나갈 구체적인 계획을 잡는다. [docs/PRD.md](PRD.md) 5절("비주얼 및
사운드 개괄")의 상위 방향("픽셀아트에 가까운 심플한 2D 그래픽")을 실제 코드 수준의
세부 규칙으로 구체화한 문서다.

## 1. 렌더링 방식 원칙 (기존 확정 사항)

[docs_temp/Phase0.md](../docs_temp/Phase0.md)에서 정한 원칙을 그대로 유지한다.

- **게임 플레이 화면**: `<canvas>` 2D context로 매 프레임 직접 그린다 (React state
  리렌더링 없이 `requestAnimationFrame` 루프에서 처리).
- **메뉴/UI 화면**(메인 화면, 종료 후 버튼 등): 일반 React DOM/CSS.
- 논리 해상도는 `480 x 640`(`GAME_WIDTH` x `GAME_HEIGHT`)으로 고정하고, 모든 좌표는
  이 좌표계를 기준으로 계산한다.

## 2. 현재 구현된 비주얼 요소 정리

### 2.1 플레이어 (`src/game/entities/player.js`)
- 사람 형태를 원(머리) + 사각형(몸통) + 선(팔·다리)의 조합으로 그린다.
- 색상: 몸통/팔다리 `#38bdf8`(하늘색), 머리 `#fde68a`(살구색).
- 피격 후 무적 시간 동안 `globalAlpha = 0.5`로 반투명 처리해 상태를 시각적으로
  알려준다.

### 2.2 작살 (`src/game/entities/harpoon.js`)
- 발사 지점(플레이어)부터 현재 끝까지 끊기지 않는 노란색(`#facc15`) 세로 막대로
  그려, 낚싯줄처럼 늘어나는 느낌을 준다.
- 플레이어가 이동하면 막대의 아래쪽 끝(x좌표)도 함께 따라간다.

### 2.3 풍선 (`src/game/entities/balloon.js`)
- 크기 단계(대/중/소)에 따라 반지름이 다른 원으로 그린다 (`BALLOON_RADII = [40, 26, 15]`).
- 색상은 크기 단계가 아니라 풍선 개체마다 랜덤하게 배정된다 (6가지 팔레트:
  `#ef4444`, `#3b82f6`, `#a855f7`, `#f97316`, `#ec4899`, `#14b8a6`). 분열된 풍선은
  부모의 색을 그대로 물려받아, 같은 풍선이 쪼개졌다는 것을 색으로도 알 수 있다.
- 그림자(`ctx.shadowBlur`/`shadowOffset`)로 입체감을 주고, 좌상단에 흰색 반투명
  방사형 그라디언트를 겹쳐 그려 광택(하이라이트) 효과를 낸다.

### 2.4 장애물 (`src/game/entities/obstacle.js`)
- 벽돌 무늬 사각형: 회색 모르타르 배경 위에 벽돌 색(`#b91c1c`/`#991b1b`/`#7f1d1d`)
  블록을 줄마다 절반씩 어긋나게(엇갈린 쌓기, running bond) 배치해 실제 벽돌담처럼
  보이게 한다.
- 스테이지가 로드될 때마다 화면 안 임의의 x 위치에 재배치된다
  (`GameCanvas.jsx`의 `loadStage`에서 `Math.random()`으로 x 좌표를 다시 계산).

### 2.5 아이템 (`src/game/entities/item.js`)
- 단순 사각형이 아니라 효과를 유추할 수 있는 아이콘으로 그린다.
  - 추가 목숨(`extra-life`): 하트 모양, `#facc15`(노랑)
  - 연사 강화(`rapid-fire`): 번개 모양, `#38bdf8`(하늘색)
- 낙하 중에는 사라지지 않고, 바닥에 닿는 즉시 사라진다 (규칙은
  [docs/features/rules.md](features/rules.md#아이템-규칙) 참고).

### 2.6 메인 화면 비주얼
- `src/components/PangIntroArt.jsx`: CSS Grid 기반 픽셀 블록으로 그린 풍선 모양
  인트로 아트 (`#4ade80` 단색 채움).
- `src/components/PangTitle.jsx`: "PANG" 네 글자를 4x5 픽셀 폰트 블록으로 그리고,
  글자마다 파스텔 팔레트(`#ffadad`, `#ffd6a5`, `#caffbf`, `#9bf6ff`)를 순서대로
  적용한다.

### 2.7 배경 테마 (`src/game/backgrounds.js`)
- 스테이지가 로드될 때마다 네 가지 테마(산, 호수, 성, 바다) 중 하나가 랜덤으로
  선택되어 캔버스 배경 전체(하늘 그라디언트 + 지형 실루엣)를 채운다.
  - 산: 하늘색 그라디언트 + 태양 + 회색조 삼각형 산 3개 + 초록 지면
  - 호수: 하늘 그라디언트 + 나무 실루엣 + 파란 호수 띠 + 물결선
  - 성: 노을 그라디언트 + 실루엣 성(탑 + 첨탑)
  - 바다: 하늘 그라디언트 + 태양 + 파란 바다 띠 + 파도선
- 실제 이미지 에셋 없이 canvas 도형(그라디언트, 삼각형, 사각형, 선)만으로 그린
  벡터 배경이다.

### 2.8 버튼 (`src/components/PixelButton.jsx`, `src/index.css`의 `.pixel-button`)
- 시작하기/음소거/다시하기/다음 스테이지/메인으로 버튼은 모두 공통 `PixelButton`
  컴포넌트를 사용한다.
- 픽셀아트 느낌의 스타일: 굵은 검은 테두리, 둥근 모서리 없음(`border-radius: 0`),
  오프셋 박스 섀도(`4px 4px 0 #111`)로 "떠 있는 버튼" 느낌을 내고, `:active` 시
  버튼이 그림자만큼 이동해 눌리는 것처럼 보인다.

### 2.9 HUD 및 화면 오버레이 (`src/components/GameCanvas.jsx`)
- 상시 HUD(좌상단): 밝은 배경 테마에서도 읽히도록 반투명 검은 패널
  (`rgba(0,0,0,0.4)`)을 먼저 깔고 그 위에 흰색 16px 텍스트로 스테이지 이름 → 점수 →
  목숨 → 남은 시간을 세로로 나열한다.
- 종료 상태 오버레이: 화면 전체를 `rgba(0,0,0,0.6)`로 어둡게 덮은 뒤 중앙에 큰
  텍스트 표시.
  - 게임 오버: `#f87171`(빨강) "GAME OVER"
  - 스테이지 클리어: `#4ade80`(초록) "STAGE CLEAR" + 카운트다운 문구
  - 일시정지: 흰색 "PAUSED" (같은 어둡게 덮는 오버레이 재사용)

분산되어 있어도 유지보수에 문제가 없었지만, 색상 종류가 계속 늘고 있으므로
4절에서 공유 팔레트 모듈화를 계속 검토 항목으로 남겨둔다.

## 4. 다음 단계 계획: 공유 팔레트 모듈화

- `src/game/palette.js`(신규)를 만들어 위 표의 색상을 의미 기반 이름으로 export한다
  (예: `COLOR_PLAYER_BODY`, `COLOR_BALLOON_PALETTE`, `COLOR_BRICK_PALETTE`,
  `COLOR_DANGER`(게임 오버), `COLOR_SUCCESS`(클리어) 등).
- 각 엔티티 렌더링 함수(`renderPlayer`, `renderBalloons`, `renderHarpoons`,
  `renderObstacle`, `renderItems`)와 `GameCanvas`의 HUD/오버레이 색상 지정 부분을
  하드코딩된 hex 값 대신 `palette.js`의 상수를 참조하도록 교체한다.
- 이렇게 하면 이후 다크/라이트 테마나 색맹 접근성 대응이 필요해졌을 때 팔레트
  모듈 하나만 바꾸면 되도록 만든다. (지금 당장 테마 전환 기능을 만들자는 것은
  아니고, 색상 값이 여러 파일에 흩어져 있는 현재 구조를 정리하는 것이 목적이다.)

## 5. 다음 단계 계획: 애니메이션/피드백 보강

현재는 상태 변화가 있어도 즉각적인 색/크기 변화 외에 별도 애니메이션이 없다.
다음을 우선순위로 검토한다.

1. **풍선 분열 이펙트**: 분열되는 순간 원래 위치에 짧게 확산되는 원(파티클 몇 개)을
   그려 "터지는" 느낌을 추가한다. `balloon.js`의 `splitBalloon` 호출 시점에
   `GameCanvas`가 짧은 수명의 이펙트 엔티티를 별도로 추가하는 방식을 검토한다.
2. **아이템 획득 반짝임**: 아이템을 먹는 순간 짧게 밝아졌다 사라지는 페이드 아웃을
   추가해 "획득했다"는 피드백을 강화한다.
3. **플레이어 피격 시 화면 흔들림(스크린 셰이크)**: 목숨이 깎일 때 캔버스를 몇
   프레임 동안 미세하게 흔들어 타격감을 준다. 캔버스 전체를 그리기 전에 `ctx.translate`를
   임의의 작은 오프셋만큼 적용하는 정도로 구현 가능하다.
4. 위 세 가지는 모두 "있으면 좋은" 폴리싱 항목이며, 게임 플레이 자체에는 영향을
   주지 않으므로 여유가 있을 때 하나씩 추가한다.

## 6. 구현됨: 스테이지별 배경 테마 (`src/game/backgrounds.js`)

- [docs/PRD.md](PRD.md) 3.4절("스테이지는 세계 각지의 유명 장소를 테마로 구성")에
  따라, 산/호수/성/바다 4가지 벡터 배경 테마를 만들고 스테이지가 로드될 때마다
  랜덤으로 하나를 선택하도록 구현했다 (2.7절 참고).
- 실제 이미지 에셋 없이 canvas 도형만으로 그렸다. 향후 실제 배경 일러스트 에셋이
  준비되면 `renderBackground(ctx, theme)`의 각 테마 함수만 이미지 렌더링으로
  교체하면 되도록 인터페이스를 분리해두었다.
- 테마를 스테이지 데이터에 고정하지 않고 매번 랜덤으로 뽑기 때문에, 같은 스테이지도
  플레이할 때마다 다른 배경으로 보일 수 있다 (요청 사항: "랜덤하게").

## 7. 범위 메모

- 이 문서는 **비주얼 계획**만 다루며, 게임 규칙/물리 로직 변경은 포함하지 않는다.
- 6절(배경 테마), 2.4절(벽돌 장애물), 2.3절(풍선 색상/그림자), 2.8절(픽셀 버튼)은
  구현이 완료되었다. 4절(팔레트 모듈화)은 순수 리팩터링이라 언제든 안전하게 적용할
  수 있고, 5절(애니메이션/피드백)은 선택적 폴리싱이므로 우선순위와 일정은 별도로
  논의한다.
