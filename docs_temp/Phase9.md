# Phase 9 — 폴리싱 및 확장 검토 (상세 구현 계획)

[docs_temp/Phases_goal.md](Phases_goal.md)의 Phase 9 목표(난이도 밸런싱, 확장 요소
재검토)에 더해, Phase 8에서 다음 단계로 미뤄두었던 항목들
([docs_temp/Phase8.md](Phase8.md) 7절 참고 — 재시작/메인 복귀 버튼, 실제 음악/효과음
에셋 교체)을 이번 Phase에서 마무리한다.

## 1. 재시작 / 메인 복귀 흐름

- Phase 8까지는 게임 오버/클리어 시 캔버스 안에 텍스트만 표시되고 상호작용 수단이
  없었다. `App.jsx`가 화면 전환을 담당하고 있으므로(Phase 1), 그 구조를 그대로 확장한다:
  - `App.jsx`에 `gameKey` 상태를 추가하고, `GameCanvas`에 `key={gameKey}`를 부여한다.
    `key`가 바뀌면 React가 컴포넌트를 완전히 새로 마운트하므로, `GameCanvas`
    내부의 모든 로컬 상태(목숨/점수/풍선 등)가 자연스럽게 초기화된다 — 별도의
    "리셋 함수"를 만들 필요가 없다.
  - `GameCanvas`는 `onExit`(메인 화면으로), `onRestart`(같은 미션 재시작) 두 prop을
    받는다. `onRestart`는 `App.jsx`에서 `gameKey`를 증가시키는 함수를 전달한다.
- `GameCanvas` 내부에 React 상태 `status`(`'playing' | 'gameOver' | 'cleared'`)를
  추가한다. 기존에는 게임 루프 클로저 안의 일반 변수(`gameOver`, `cleared`)로만
  종료 조건을 추적했는데, 이는 캔버스 렌더링 판단에는 충분하지만 DOM 버튼을
  조건부로 보여주려면 React 리렌더링이 필요하다. 상태가 `false → true`로 바뀌는
  바로 그 프레임에 `setStatus(...)`를 한 번만 호출해 불필요한 리렌더링을 막는다
  (매 프레임 호출하지 않음 — 기존의 "한 번만 재생" 효과음 로직과 동일한 엣지 감지
  패턴을 재사용).
- `status !== 'playing'`일 때 캔버스 아래에 "다시하기"/"메인으로" 버튼을 표시한다.

## 2. 실제 음악/효과음 에셋 교체 — 재검토 결과: **보류**

- Phase 8에서 만든 `src/game/audio.js`의 Web Audio 합성 SFX/BGM은 외부 파일 없이
  동작하며 시스템 요구사항(효과음 존재, 배경음 존재, 음소거 가능)을 이미 충족한다.
- 실제 음악/효과음 에셋(mp3 등)을 적용하려면 저작권이 확실한 리소스 파일을
  프로젝트에 추가해야 하는데, 이번 세션에서는 사용할 에셋 파일이 준비되어 있지
  않다. 임의의 외부 URL을 추측해 가져오는 것은 지양해야 하므로, **이번 Phase에서는
  실제 에셋 교체를 진행하지 않고 현재의 합성 사운드를 유지**하기로 한다.
- 향후 실제 에셋(파일)이 프로젝트에 추가되면, `audio.js`의 `playTone` 기반 함수들을
  `new Audio(url).play()` 방식으로 교체하기만 하면 되도록 이미 함수 시그니처가
  분리되어 있다 (Phase 8에서 의도한 설계).

## 3. 난이도 밸런싱

- `docs/features/mission.md`의 "풍선 구성: 큰 풍선 1~2개로 시작"에 맞춰, 현재
  하드코딩된 큰 풍선 1개를 2개로 늘리고 서로 다른 위치/좌우 속도로 시작하게 해
  처음부터 화면이 밋밋하지 않도록 조정한다 (`src/game/stages/mission1.js`).
- 그 외 물리 상수(중력, 반사 속도, 분열 속도)는 Phase 3~7 동안 플레이 테스트로
  검증된 값이므로 이번 Phase에서 임의로 바꾸지 않는다. 실제 플레이 피드백이
  누적되면 그때 조정한다.

## 4. 확장 요소 재검토 — 멀티플레이어

- PRD [3.7 멀티플레이어](../docs/PRD.md) 및 범위 메모에 "싱글플레이만 구현"이라고
  명시되어 있고, 지금까지 구현된 입력/충돌/스테이지 시스템은 모두 플레이어 1명을
  전제로 설계되어 있다 (예: `player` 단일 객체, 입력 모듈이 전역 키 상태 하나만
  관리).
- 2인 협동을 지원하려면 플레이어 배열화, 입력 매핑 분리(P1/P2 키 셋), 충돌 판정을
  플레이어별로 반복하는 등 여러 시스템에 걸친 리팩터링이 필요하다. 현재 범위(1개
  스테이지, 싱글 플레이 검증)에서는 그 비용 대비 실익이 낮다고 판단해 **이번
  릴리스에서는 구현하지 않는다**는 결론을 유지한다 (재검토 결과: 보류).
- 이후 스테이지가 늘어나고 싱글플레이 경험이 안정화된 뒤, 별도 Phase로 분리해
  검토하는 것을 권장한다.

## 5. 완료(Done) 기준

- 게임 오버 또는 스테이지 클리어 후 "다시하기" 버튼을 누르면 같은 미션이 처음부터
  다시 시작된다 (목숨/점수/풍선 상태 초기화).
- "메인으로" 버튼을 누르면 메인 화면으로 돌아가고, 배경음이 멈춘다.
- Mission 1 시작 시 큰 풍선 2개가 서로 다른 위치에서 시작한다.
- 콘솔 에러 없음.

## 6. Phase 9 이후 연결

- 이 시점 이후는 [docs_temp/Phases_goal.md](Phases_goal.md)에 정의된 9개 Phase가
  모두 완료된 상태다. 추가 스테이지, 실제 에셋, 멀티플레이어 등은 신규 Phase 계획
  문서를 별도로 작성해 진행한다.
