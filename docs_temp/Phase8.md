# Phase 8 — UI/UX 다듬기 (상세 구현 계획)

[docs_temp/Phases_goal.md](Phases_goal.md)의 Phase 8 목표 세 가지를 구현한다:
점수/목숨/스테이지 표시 UI, 기본 사운드 효과/BGM, 게임 오버/클리어 화면 다듬기.
지금까지는 목숨/시간만 HUD에 표시했고 점수 시스템 자체가 없었으므로, 점수 시스템을
새로 추가하는 것이 이번 Phase의 핵심 작업 중 하나다. 오디오는 별도의 사운드/음악
에셋 파일이 없으므로, **Web Audio API로 즉석에서 톤(tone)을 합성**하는 방식으로
구현한다 (외부 파일 의존 없이 동작).

## 목표

- 풍선을 맞출 때마다 점수가 오르고, HUD에 점수/목숨/스테이지 이름이 함께 표시된다.
- 주요 이벤트(발사, 풍선 명중, 아이템 획득, 게임 오버, 스테이지 클리어)에 짧은
  효과음이 재생되고, 게임 플레이 중에는 은은한 배경음(BGM 루프)이 재생된다.
- 음소거 토글 버튼을 제공한다 (오디오는 호불호가 갈리므로 항상 켤 수 있게 강제하지
  않는다).
- 게임 오버/클리어 화면에 최종 점수를 함께 표시해 화면 구성을 보강한다.

## 1. 점수 시스템

- `constants.js`에 `SCORE_PER_HIT = [100, 150, 200]` (대/중/소 순으로, 맞히기
  어려운 작은 풍선일수록 높은 점수)를 추가한다.
- `GameCanvas.jsx`에 `score` 로컬 변수를 추가하고, 작살-풍선 충돌 처리 시
  `score += SCORE_PER_HIT[balloon.stage]`로 누적한다 (분열 여부와 무관하게 맞힐 때마다
  적립).
- HUD에 `Score: N`을 목숨/시간과 함께 표시한다.

## 2. 스테이지 이름 표시

- `src/game/stages/mission1.js`의 `MISSION_1`에 `name: 'Mission 1'` 필드를 추가한다.
- HUD 상단에 스테이지 이름을 표시한다. 이후 스테이지가 늘어나도 각 스테이지 데이터가
  자신의 이름을 들고 있으므로 `GameCanvas`는 `MISSION_1.name`만 참조하면 된다.

## 3. 사운드 (`src/game/audio.js`, 신규 모듈)

브라우저의 자동재생 정책상 `AudioContext`는 **사용자 제스처 안에서** 생성/재개되어야
하므로, 메인 화면의 "시작하기" 버튼 클릭 시점에 초기화한다.

- `initAudio()`: `AudioContext`를 생성(최초 1회)하고 필요 시 `resume()`한다.
  `MainScreen`의 시작하기 버튼 클릭 핸들러에서 호출한다.
- `playTone({ frequency, duration, type, volume, startDelay })`: 오실레이터 하나를
  생성해 지정한 주파수/길이/파형으로 짧은 톤을 재생하는 저수준 유틸리티.
- 이를 기반으로 한 이벤트별 SFX 함수:
  - `playFireSound()` — 작살 발사 (높은 톤, 아주 짧게)
  - `playPopSound(stage)` — 풍선 명중 (풍선 단계에 따라 음높이를 다르게)
  - `playItemSound()` — 아이템 획득
  - `playGameOverSound()` — 낮은 톤의 게임 오버 사운드
  - `playClearSound()` — 두 음을 순서대로 재생하는 간단한 클리어 징글
- `startBgm()` / `stopBgm()`: `setInterval`로 몇 개의 음을 낮은 볼륨으로 순환
  재생하는 아주 단순한 루프 (정식 음악 에셋이 준비되면 이 함수들의 구현만 교체하면
  되도록 인터페이스를 분리해둔다).
- `setMuted(value)` / `isMuted()`: 전역 음소거 상태. 음소거 시 `playTone`이 아무
  것도 재생하지 않는다.

## 4. 이벤트 연결

- `harpoon.js`의 `spawnHarpoon`에서 작살이 실제로 생성되는 순간 `playFireSound()`를
  호출한다 (발사 시점의 사이드 이펙트이므로 발사 로직 바로 옆에 두는 것이 자연스럽다).
- `GameCanvas.jsx`의 `update(dt)`에서:
  - 작살-풍선 충돌이 확인된 프레임에 `playPopSound(balloon.stage)` 호출
  - 아이템을 주운 프레임에 `playItemSound()` 호출
  - `gameOver`/`cleared`가 **false → true로 바뀌는 그 프레임에만** 각각
    `playGameOverSound()` / `playClearSound()`를 한 번 호출한다 (매 프레임 반복
    재생되지 않도록 상태 전이 시점을 명확히 확인).
- `GameCanvas.jsx`의 `useEffect`에서 마운트 시 `startBgm()`, 언마운트 cleanup에서
  `stopBgm()`을 호출해 화면 전환 시 배경음이 계속 겹쳐 재생되지 않게 한다.

## 5. 음소거 토글 (`App.jsx`)

- 화면 전환과 무관하게 항상 보이도록 `App.jsx` 최상단에 작은 음소거 토글 버튼을
  추가한다 (우측 상단 고정 위치).
- 버튼은 `audio.js`의 `setMuted`/`isMuted`를 감싸는 로컬 `useState`로 아이콘을
  갱신한다 (🔊 / 🔇).

## 6. 게임 오버 / 클리어 화면 다듬기

- 기존에는 텍스트만 표시했는데, 다음을 보강한다:
  - 배경을 반투명 검은 사각형으로 살짝 어둡게 덮어 텍스트 가독성을 높인다.
  - "GAME OVER" / "STAGE CLEAR" 아래에 최종 점수(`Score: N`)를 함께 표시한다.
- 재시작/메인으로 돌아가기 버튼은 화면 상태 관리 리팩터링이 필요해 범위가 커지므로
  **이번 Phase에서는 다루지 않고 Phase 9(폴리싱)에서 재검토**한다 (화면 구성/피드백
  개선에 집중하고, 인터랙션 흐름 변경은 다음 단계로 미룬다).

## 7. 완료(Done) 기준

- HUD에 `Mission 1`, `Score`, `Lives`, `Time`이 모두 표시된다.
- 발사/풍선 명중/아이템 획득/게임 오버/스테이지 클리어 시 각각 다른 효과음이 들린다.
- 게임 화면 진입 후 은은한 배경음이 반복 재생되고, 메인 화면으로 돌아가거나
  컴포넌트가 언마운트되면 멈춘다.
- 음소거 버튼을 누르면 모든 효과음/배경음이 즉시 멈추고, 다시 누르면 복구된다.
- 게임 오버/클리어 화면에 어두운 오버레이와 최종 점수가 표시된다.
- 콘솔 에러 없음.

## 8. Phase 8 이후 연결

- Phase 9(폴리싱)에서 재시작/메인 복귀 버튼, 실제 음악/효과음 에셋 교체, 난이도
  밸런싱을 재검토한다.
- 스테이지가 여러 개로 늘어나면 `SCORE_PER_HIT`처럼 전역 상수로 둔 값들을 스테이지별
  난이도에 맞게 조정할지 재검토한다.
