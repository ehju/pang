# Phase 6 — 게임 룰 마무리 (상세 구현 계획)

[docs_temp/Phases_goal.md](Phases_goal.md)의 Phase 6 목표는 두 가지다: (1)
[docs/features/rules.md](../docs/features/rules.md)의 승리/패배 조건이 실제 구현과
일치하는지 정리·검증하고, (2) 제한 시간 및 페널티 로직을 이번 초안 범위에 포함할지
재확인한 뒤 필요한 만큼만 구현하는 것이다.

## 1. 승리/패배 조건 정리 및 검증

`rules.md`의 조건과 현재 구현을 대조한다.

| 규칙(rules.md) | 구현 위치 | 상태 |
|---|---|---|
| 스테이지 클리어: 모든 풍선 제거 시 다음 진행 | `GameCanvas.jsx`의 `cleared` (Phase 5) | ✅ 구현됨 |
| 목숨 감소: 풍선에 닿으면 목숨 1 감소 | `damagePlayer` (Phase 3) | ✅ 구현됨 |
| 게임 오버: 목숨 모두 소진 | `GameCanvas.jsx`의 `gameOver` (Phase 3) | ✅ 구현됨 |

- 세 조건 모두 이미 구현되어 있고 Phase 3/5에서 동작을 확인했으므로, 이번 Phase에서
  로직 자체를 새로 만들 필요는 없다.
- 다만 두 상태(`gameOver`, `cleared`)가 동시에 참이 될 수 없다는 것을 명확히 한다:
  풍선이 모두 사라진 프레임에는 플레이어가 더 이상 피격될 수 없으므로 상호 배타적
  관계가 자연스럽게 유지된다 (Phase 5에서 이미 `if/else`로 렌더링 분기 처리함).
  이번 Phase에서는 이 불변조건이 시간 제한 페널티 도입 이후에도 깨지지 않는지만
  재확인한다 (아래 3번 참고).

## 2. 제한 시간 및 페널티: 적용 여부 재확인 → **최소 범위로 적용**

- `rules.md`에 "스테이지별 제한 시간이 존재하며, 시간 내 클리어하지 못하면 페널티
  (풍선 추가 생성 등)가 발생할 수 있다"고 명시되어 있고, `mission.md`도 "제한 시간:
  넉넉하게 설정"이라고 Mission 1의 조건을 구체적으로 언급하고 있다. PRD 범위
  메모("세부 스펙보다는 개괄" 등)와 상충하지 않는 선에서, **제한 시간 + 페널티(풍선
  추가 생성)를 최소 형태로 구현**하기로 한다.
- 구체 수치(제한 시간 길이, 페널티 강도)는 정해진 바 없으므로, `mission.md`의
  "넉넉하게 설정" 지침에 따라 Mission 1에서는 체감상 거의 발동하지 않을 정도로
  긴 값(예: 120초)을 사용한다.

## 3. 구현 방식

- 스테이지 데이터(`src/game/stages/mission1.js`)에 두 필드를 추가한다:
  - `timeLimit`: 제한 시간(초)
  - `timePenaltyBalloon`: 시간 초과 시 추가로 스폰할 풍선의 초기 상태(Phase 5의
    풍선 데이터와 동일한 형식) — 스테이지 데이터가 "이 스테이지에서 등장 가능한
    모든 풍선의 형태"를 기술한다는 기존 원칙을 유지한다.
- `GameCanvas.jsx`에 `remainingTime` 로컬 상태를 추가한다:
  - `gameOver`/`cleared`가 아닐 때만 `remainingTime -= dt`
  - `remainingTime <= 0`이 되면 `timePenaltyBalloon` 데이터로 풍선을 하나 추가
    생성하고, `remainingTime`을 다시 `timeLimit`으로 리셋한다 (반복 페널티).
  - 페널티 스폰은 클리어 판정(`balloons.length === 0`) **이전에** 처리해, 시간
    초과와 마지막 풍선 제거가 같은 프레임에 겹쳐도 스테이지가 잘못 클리어되지
    않게 한다.
- HUD에 남은 시간을 표시한다 (목숨 표시 옆, 예: `Time: 87s`).

## 4. 완료(Done) 기준

- Mission 1 진입 시 제한 시간이 카운트다운되며 HUD에 표시된다.
- 제한 시간이 0이 되면 화면에 풍선이 하나 추가되고, 타이머가 다시 처음부터
  시작된다.
- 시간 초과 페널티가 발생해도 게임 오버/클리어 조건은 기존과 동일하게 정확히
  동작한다 (목숨 0 → GAME OVER, 풍선 0 → STAGE CLEAR).
- 게임 오버 또는 클리어 상태에서는 타이머가 더 이상 감소하지 않는다.
- 콘솔 에러 없음.

## 5. Phase 6 이후 연결

- Phase 7(아이템)에서 스테이지 데이터에 아이템 스폰 정보가 추가될 때도, 이번
  Phase에서 정착시킨 "스테이지 데이터가 모든 배치/이벤트를 기술한다"는 패턴을
  그대로 따른다.
- 페널티 강도(추가 풍선 개수, 크기 등)의 구체적인 밸런싱은 Phase 9(폴리싱)에서
  재검토한다.
