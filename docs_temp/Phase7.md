# Phase 7 — 아이템 시스템 (상세 구현 계획)

[docs_temp/Phases_goal.md](Phases_goal.md)의 Phase 7 목표와
[docs/features/rules.md](../docs/features/rules.md#아이템-규칙)의 아이템 규칙,
[docs/features/mission.md](../docs/features/mission.md)의 "기본 아이템 1개 정도
등장시켜 아이템 시스템을 소개" 범위를 구현한다. 확장 기능이므로 rules.md에 나열된
여러 효과 중 대표적인 두 가지(즉시 효과 1개, 지속 시간형 효과 1개)만 우선
구현하고, 나머지(무적, 특수 무기 등)는 이후 확장 여지로 남긴다.

## 목표

- 풍선이 작살에 맞을 때 확률적으로 아이템이 드롭된다.
- 아이템은 바닥까지 서서히 떨어진 뒤 일정 시간이 지나면 사라진다.
- 플레이어가 아이템에 닿으면 효과가 적용되고 아이템은 사라진다.
- 두 가지 효과를 구현한다: **추가 목숨**(즉시 효과), **연사 강화**(지속 시간형 효과 —
  쿨타임 단축).

## 1. 아이템 엔티티 (`src/game/entities/item.js`)

- 상태: `{ x, y, type, timeRemaining }`
- `updateItems(items, dt)`:
  - 바닥(`GROUND_Y`)에 닿기 전까지는 `ITEM_FALL_SPEED`로 서서히 하강하고, 바닥에
    닿으면 그 자리에 멈춰 남은 시간 동안 유지된다 (원작처럼 주울 때까지 바닥에
    놓여있는 느낌).
  - `timeRemaining -= dt`, `timeRemaining <= 0`이 되면 배열에서 제거된다
    (rules.md: "일정 시간 내에 습득하지 않으면 사라진다").
- `renderItems(ctx, items)`: 타입별로 다른 색 사각형으로 표시 (추가 목숨=노랑,
  연사 강화=하늘색).

## 2. 드롭 로직

- 작살이 풍선에 맞는 순간(분열 여부와 무관하게)마다 `ITEM_DROP_CHANCE` 확률로
  아이템 하나를 그 풍선의 위치에 생성한다.
- 아이템 종류는 두 가지 중 무작위로 하나 선택한다 (`extra-life`, `rapid-fire`).
- 스테이지별로 확률을 다르게 주는 것은 아직 필요하지 않으므로(스테이지가 1개뿐),
  드롭 확률은 전역 상수로 관리한다. 스테이지가 늘어나 난이도별 조정이 필요해지면
  그때 스테이지 데이터로 옮긴다 (Phase 5에서 정착시킨 패턴 재사용 가능).

## 3. 효과 적용

- **추가 목숨 (`extra-life`)**: 즉시 `player.lives += 1`. 지속시간 없음.
- **연사 강화 (`rapid-fire`)**: 작살 시스템에 `rapidFireRemaining` 타이머를 추가해
  일정 시간(`RAPID_FIRE_DURATION`) 동안 발사 쿨타임을 `HARPOON_COOLDOWN_RAPID`
  (더 짧은 값)로 대체한다. 시간이 지나면 원래 쿨타임으로 자동 복귀한다.
- 두 효과 모두 이후 다른 아이템(이동 속도 증가, 일시 무적 등)을 추가할 때 같은
  패턴(즉시 효과 vs 타이머 기반 지속 효과)을 재사용할 수 있도록 설계하되, 지금
  당장 필요하지 않은 효과 종류를 미리 만들어두지는 않는다.

## 4. 충돌 판정

- `src/game/systems/collision.js`에 `playerHitsItem(player, item)`을 추가한다.
  아이템은 작은 사각형이므로 플레이어 사각형과의 단순 AABB 겹침 판정을 사용한다
  (풍선처럼 원-사각형 근사가 필요 없음).

## 5. GameCanvas 통합

- `update(dt)` 순서에 추가:
  1. 작살-풍선 충돌 처리 시, 맞은 풍선 위치에서 `ITEM_DROP_CHANCE` 판정 후 필요 시
     아이템 생성
  2. `updateItems(items, dt)`
  3. 플레이어-아이템 충돌 검사 → 효과 적용 후 아이템 제거
- 렌더링 순서: 배경 → 장애물 → 풍선 → 아이템 → 작살 → 플레이어 (아이템은 풍선보다
  위, 작살/플레이어보다는 아래에 그려 자연스럽게 겹치게 한다).

## 6. 완료(Done) 기준

- 풍선을 맞출 때마다 일정 확률로 아이템이 나타나 바닥까지 떨어진 뒤 멈춘다.
- 아이템을 줍지 않고 일정 시간이 지나면 사라진다.
- 플레이어가 "추가 목숨" 아이템을 먹으면 즉시 목숨이 1 증가한다 (HUD에 반영).
- 플레이어가 "연사 강화" 아이템을 먹으면 일정 시간 동안 스페이스바 연사 간격이
  짧아지고, 시간이 지나면 원래대로 돌아온다.
- 콘솔 에러 없음.

## 7. Phase 7 이후 연결

- rules.md에 예시로 언급된 나머지 효과(이동 속도 증가, 일시 무적, 특수 무기)는
  필요성이 확인되면 같은 `item.js` / 효과 적용 패턴으로 추가한다.
- 스테이지가 여러 개로 늘어나면 드롭 확률/아이템 종류를 스테이지 데이터로 옮기는
  것을 재검토한다 (Phase 6에서 시간 제한을 스테이지 데이터화한 것과 동일한 방향).
