# Phase 3 — 풍선 물리 및 분열 (상세 구현 계획)

[docs_temp/Phases_goal.md](Phases_goal.md)의 Phase 3 목표와
[docs/features/rules.md](../docs/features/rules.md)의 풍선 규칙을 구현한다. Phase 2에서
만든 플레이어/작살에 이어, 풍선 엔티티와 물리(중력, 반사), 작살 충돌 시 분열, 플레이어
충돌 시 목숨 감소까지 다룬다. 스테이지 데이터(어떤 풍선을 몇 개, 어디서 시작할지)는
Phase 5에서 다루므로, 이번 Phase에서는 테스트용으로 몇 개의 큰 풍선을 하드코딩해
동작을 검증한다.

## 목표

- 풍선이 중력의 영향을 받아 포물선 궤적으로 움직이고, 바닥/벽에서 반사(튕김)된다.
- 작살에 맞은 풍선은 크기 단계에 따라 분열(대→중→소) 또는 소멸(소→없음)한다.
- 풍선이 플레이어와 충돌하면 목숨이 감소한다 (연속 감소 방지용 짧은 무적 시간 포함).
- 목숨이 0이 되면 게임 오버 상태로 전환된다 (화면 표시만, 재시작 로직은 범위 밖).

## 1. 풍선 엔티티 (`src/game/entities/balloon.js`)

- 상태: `{ x, y, vx, vy, stage }`
  - `stage`: `0`(대) → `1`(중) → `2`(소). `BALLOON_RADII[stage]`로 반지름을 조회한다.
- 물리 (`updateBalloons(balloons, dt)`):
  - `vy += GRAVITY * dt` (중력 가속)
  - `x += vx * dt`, `y += vy * dt`
  - 바닥 반사: `y + radius >= GROUND_Y` → `y`를 클램프하고 `vy = -Math.abs(vy)`
    (원작처럼 감쇠 없이 동일한 속도로 튕겨오르는 것을 기본값으로 하고, 이후 플레이
    테스트로 반발 계수를 조정할 수 있게 상수화한다)
  - 벽 반사: `x - radius <= 0` 또는 `x + radius >= GAME_WIDTH` → 위치 클램프 후
    `vx = -vx`
  - 천장은 별도 처리하지 않는다 (분열 시 초기 속도를 화면 안에서 자연스럽게 포물선을
    그리는 값으로 설정해 천장에 닿지 않도록 한다. 필요 시 이후 조정).
- 분열/소멸 로직:
  - `splitBalloon(balloon)`: `stage < 2`이면 같은 위치에서 `stage + 1`짜리 풍선 2개를
    생성해 반환 (하나는 `vx`가 양수, 하나는 음수 — 좌우로 갈라지는 느낌). `vy`는
    분열 순간 위로 튀는 초기값(`BALLOON_SPLIT_VY`)으로 재설정한다.
  - `stage === 2`(가장 작은 풍선)면 분열 결과 없이 제거된다.
- `renderBalloons(ctx, balloons)`: 단계별로 다른 색(예: 대=진한 초록, 중=초록,
  소=연두)으로 원을 그린다.

## 2. 상수 추가 (`constants.js`)

- `GRAVITY` (px/s², 예: 400)
- `BALLOON_RADII = [40, 26, 15]` (대/중/소)
- `BALLOON_SPLIT_VY` (분열 시 위로 튀는 초기 속도, 음수 값)
- `BALLOON_SPLIT_VX` (분열된 두 풍선의 좌우 속도 크기)
- `PLAYER_LIVES_INITIAL` (예: 3)
- `PLAYER_INVULNERABLE_TIME` (풍선 충돌 후 무적 시간, 예: 1.5초 — 짧은 시간 내
  중복으로 목숨이 깎이는 것을 방지)

## 3. 충돌 판정 (`src/game/systems/collision.js`)

풍선은 원(circle), 작살은 얇은 세로선, 플레이어는 사각형이므로 각각 다른 근사
판정을 사용한다.

- **작살 vs 풍선**: 작살은 `harpoon.x` 근처의 세로선이므로, 풍선 중심과의 수평
  거리가 `radius` 이내이고, 풍선의 수직 범위가 작살의 `[y, startY]` 구간과
  겹치면 충돌로 판정한다.
- **플레이어 vs 풍선**: 사각형-원 충돌의 표준 근사인 "사각형에서 원 중심에 가장
  가까운 점을 구해 그 점과의 거리가 반지름 이내인지" 방식을 사용한다.
- 이 모듈은 순수 함수로 작성해 GameCanvas의 update 루프에서 매 프레임 호출한다.

## 4. GameCanvas 통합 흐름 (매 프레임)

1. `updatePlayer`, `updateHarpoons` (Phase 2와 동일)
2. `updateBalloons` (중력/반사)
3. 작살-풍선 충돌 검사 → 맞은 작살 제거 + 맞은 풍선을 분열 결과로 교체
4. 플레이어-풍선 충돌 검사 (무적 시간이 0일 때만) → 목숨 감소, 무적 시간 리셋
5. 목숨이 0 이하이면 게임 오버 플래그 설정 (이후 루프에서 입력 무시, 캔버스에
   "GAME OVER" 텍스트 표시)
6. 렌더링 순서: 배경 → 풍선 → 작살 → 플레이어 → (게임 오버 시) 오버레이 텍스트

## 5. 테스트용 풍선 스폰 (임시)

- Mission 데이터가 아직 없으므로, `GameCanvas` 마운트 시 큰 풍선(stage 0) 1~2개를
  화면 상단 근처에 하드코딩으로 생성해 물리/분열/충돌을 눈으로 확인할 수 있게 한다.
- 이 하드코딩은 Phase 5(스테이지 시스템)에서 Mission 데이터 기반 스폰으로 교체될
  것임을 코드 주석 없이도 알 수 있도록, 스폰 로직을 `GameCanvas` 최상단에 짧게
  분리해둔다 (별도 모듈로 과설계하지 않는다).

## 6. 완료(Done) 기준

- 풍선이 중력을 받아 포물선을 그리며 바닥/벽에서 자연스럽게 튕긴다.
- 작살로 큰 풍선을 맞추면 중간 크기 풍선 2개로 갈라지고, 중간 풍선을 맞추면 작은
  풍선 2개로, 작은 풍선을 맞추면 사라진다.
- 풍선에 플레이어가 닿으면 목숨이 1 감소하고, 짧은 무적 시간 동안은 추가로 깎이지
  않는다.
- 목숨이 0이 되면 "GAME OVER"가 화면에 표시되고 더 이상 조작이 반영되지 않는다.
- 콘솔 에러 없음.

## 7. Phase 3 이후 연결

- Phase 4(장애물)에서 풍선의 벽/바닥 반사 로직과 동일한 패턴으로 장애물 반사를
  추가하고, 작살이 장애물에서 막히는 조건을 `collision.js`에 확장한다.
- Phase 5(스테이지 시스템)에서 하드코딩된 테스트용 풍선 스폰을 Mission 데이터
  기반 스폰으로 교체하고, "모든 풍선 제거 시 클리어" 판정을 추가한다.
- 점수 시스템, 정식 게임오버/재시작 UI는 Phase 6~8에서 다룬다.
