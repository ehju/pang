# PANG 구현 Phase별 목표

[docs/PRD.md](../docs/PRD.md), [docs/features/](../docs/features/) 문서를 기반으로
실제 구현을 위한 단계별(Phase) 목표를 정리한다. 각 Phase는 이전 Phase 위에서 동작
가능한 상태를 유지하는 것을 원칙으로 한다.

## Phase 0 — 프로젝트 기반 준비
- 게임 루프(캔버스 또는 DOM 기반) 렌더링 구조 설계
- 기본 화면 레이아웃(게임 영역 크기, 좌표계) 정의
- 키보드 입력 처리 기반 마련 (좌/우 이동, 발사 키)

## Phase 1 — 메인 화면
- [docs/features/main.md](../docs/features/main.md) 기반 타이틀 화면 구현
- 시작하기 버튼 → 게임(Mission 1) 진입 흐름 연결

## Phase 2 — 플레이어 이동 및 작살 발사
- 캐릭터 좌우 이동 구현
- 작살 발사(수직 발사, 화면 상단 또는 장애물까지 도달) 구현
- 발사 쿨타임/동시 발사 개수 등 기본 규칙 적용

## Phase 3 — 풍선 물리 및 분열
- 풍선의 중력/바닥·벽 반사 이동 구현
- 작살에 맞았을 때 크기 단계별 분열 로직 구현 (대→중→소→소멸)
- 풍선-캐릭터 충돌 시 목숨 감소 처리

## Phase 4 — 장애물
- 장애물(벽/플랫폼) 배치 및 렌더링
- 풍선이 장애물에 부딪히면 반사되는 물리 처리
- 작살이 장애물에서 막히는(관통 불가) 처리

## Phase 5 — 스테이지(Mission) 시스템
- [docs/features/mission.md](../docs/features/mission.md) 기반 Mission 1 스테이지 데이터 구성
- 모든 풍선 제거 시 스테이지 클리어 판정
- 목숨 소진 시 게임 오버 처리

## Phase 6 — 게임 룰 마무리
- [docs/features/rules.md](../docs/features/rules.md) 기반 승리/패배 조건 정리 및 검증
- 제한 시간 및 페널티 로직 (초안 범위 내에서 적용 여부 재확인)

## Phase 7 — 아이템 시스템 (확장)
- 풍선 제거 시 아이템 드롭 로직
- 아이템 습득 및 효과 적용 (작살 강화, 이동 속도 등)
- 아이템 미습득 시 소멸 처리

## Phase 8 — UI/UX 다듬기
- 점수/목숨/스테이지 표시 UI
- 사운드 효과 및 기본 BGM 적용
- 게임 오버 / 클리어 화면 구성

## Phase 9 — 폴리싱 및 확장 검토
- 난이도 밸런싱 (풍선 속도/개수, 스테이지 진행)
- 멀티플레이어 등 확장 요소 재검토 (PRD 6번 범위 메모 참고)

## 진행 원칙
- 각 Phase는 독립적으로 눈으로 확인 가능한 결과물을 목표로 한다.
- 세부 스펙이 확정되지 않은 항목(아이템 효과 수치, 난이도 곡선 등)은 해당 Phase
  착수 시점에 별도 문서로 구체화한다.
